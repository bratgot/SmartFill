# Licensing of nuke-ai-fill

Quick-reference for studio legal teams. See THIRD_PARTY_LICENSES.md
for full attribution and license texts.

## Bottom line

**Commercial use is permitted.** nuke-ai-fill is MIT-licensed. Every
bundled software component is MIT, Apache 2.0, BSD-3-Clause, or
Public Domain -- all permissive open-source licenses that allow
proprietary commercial use, modification, and redistribution.

Studios can use the plugin on paying client work, ship VFX shots
made with it, and modify the source for in-house pipelines.

## Critical warning: do NOT use FLUX.1-dev

The plugin defaults to FLUX.1-schnell (Apache 2.0) for text-to-image
generation. Black Forest Labs releases multiple FLUX models with
very similar names:

| Model | License | Commercial use? |
|---|---|---|
| FLUX.1-schnell | Apache 2.0 | YES, free |
| FLUX.1-dev | FLUX.1 [dev] Non-Commercial License v1.1.1 | NO - requires paid license from BFL |
| FLUX.1-pro, FLUX.1-fill-dev, canny-dev, depth-dev, etc. | Various non-commercial / proprietary | NO unless licensed |

The plugin's `Diffusion Model` knob accepts any FLUX GGUF file
that loads in stable-diffusion.cpp. If an artist swaps in a [dev]
model file because they prefer the output quality, the studio is
suddenly out of compliance with that model's license -- which
includes "revenue-generating activities, direct interactions with
or impact on end users."

**Mitigation**: in your studio pipeline, restrict the plugin's
models directory to contain only Apache 2.0 FLUX files. The
recommended file is `flux1-schnell-q4_0.gguf` from
huggingface.co/leejet/FLUX.1-schnell-gguf.

## Attribution obligations when redistributing

If your studio redistributes the plugin (e.g. to subcontractors,
internal team members at multiple sites, or as part of a derivative
tool):

1. Include the LICENSE file (MIT for nuke-ai-fill)
2. Include THIRD_PARTY_LICENSES.md (lists all upstream attributions)
3. Optionally, include this LICENSING.md as the legal summary
4. If you bundle model files, the Apache 2.0 text in
   THIRD_PARTY_LICENSES.md serves as the License copy required
   by Apache 2.0 section 4(a)

The `package.ps1` distribution script automatically includes all
three files in the output zip.

## Outputs

Images generated by this plugin are not subject to any licensing
restriction from the licenses above. Per FLUX.1-schnell's terms,
"the model can be used for personal, scientific, and commercial
purposes." Per Apache 2.0 and MIT in general, outputs of running
the software are unrestricted.

Copyright in the generated images themselves is a separate matter
(jurisdiction-specific, evolving). This document does not constitute
legal advice on output copyright.

## Output copyright disclosure

If your studio's client contracts require disclosure of AI-generated
content, you may wish to log which shots used AIGenerate or
AISmartFill. The plugin does not currently do this automatically.
A future addition could write provenance metadata into the output.

## Patent

Apache 2.0 includes an explicit patent grant from contributors, with
a defensive termination clause: if you sue a contributor over patents
related to the Work, your license terminates. This is standard
practice in Apache projects (TensorFlow, Kubernetes, Kafka, etc.)
and is not a barrier to commercial deployment.

MIT does not include an explicit patent grant. The de-facto practice
is that distributing under MIT carries an implied patent license to
the extent the patents are necessary to use the code as distributed.
